# Report 0089 — The continue-beat — ambient auto-continuation, silent on escalation

## What landed

**Done-line 0127 — the continue-beat** (PR #236, branch `claude/continue-beat-patrol`, stacked on #226). bdo asked for an ambient trigger so that when he is here but AFK, a session keeps working without being re-prompted, stopping when it actually needs him and going silent the moment it escalates. The shape was confirmed through the `/ask` panel before any code: build the in-session beat now (a `Stop` hook continues a *live* session — it cannot wake a process that already exited; the timed cloud-wake is a separate deferred layer); silence on needs-you / parked-at-stamp / explicit-escalate plus a safety ceiling; and an empty backlog does **not** silence the patrol (his deliberate choice — the idle default is to find the next work).

- `loop/patrol.py` — a pure `decide(state, session_id, gap, owner_present)` fold + a write-once `escalate` verb. It **composes** `loop.gaps.top_gap` and `loop.summon.owner_backlog` (imports them, never re-derives — I-4). CONTINUE when there is self-serviceable work and no escalation; SILENT on escalate-marker / ceiling(12) / parked-at-stamp (gap absent **and** owner backlog non-empty — a non-empty owner queue alone never silences). The escalate marker is gitignored nag-state (`.ai-native/continue-beat.json`, the `mock-shame.json` precedent), never the truth log.
- `.claude/hooks/continue_beat.py` — the `Stop` hook, wired beside `reflect_auto`. Prints `{"decision":"block","reason":<next move>}` to continue, nothing to stop. It is **fail-SAFE-to-STOP**: any internal error or malformed payload lets the session stop (the opposite of the other beats' fail-open), because a bug that always-continued would trap an AFK session burning tokens.
- `tests/test_patrol.py` — the §10 teeth: a constant decider fails (has-work must continue AND escalated must go silent — both proven reachable; the ceiling, parked-at-stamp, new-session-reset, and the hook's fail-safe each pinned, two by subprocess). Full suite green at **918**.
- Backed by `atom.continue-beat.v0` with an **independent** value-gate receipt `rcp.04a1f8626170` (D-2): a separate branded agent (`ontum-node:value-gate.claude.v1`) read the work, ran the tests itself (15 OK), verified the fail-safe direction and the teeth, and accepted.
