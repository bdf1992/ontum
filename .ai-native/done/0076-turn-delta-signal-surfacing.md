> **Done when:** A **signal registry** — config-as-code, one declared versioned source (prompt-as-code, *not* literals scattered in the hook) — declares each watched signal: its identity, how its count folds from the log, and **the action it carries** (the pen/skill/command to run, the thing to read or execute). Watched signals at minimum: open summons, owner-stamp backlog, surface-drift, and bdo's inbound GitHub gestures; `gesture_surface.py` is wired to **UserPromptSubmit** so the inbound sensor fires every turn (the cadence done-line 0044 named and deferred). A gitignored last-seen snapshot (the `mock_shame` nag-state pattern — a pure cache under `.ai-native/`, never truth) records the count each signal was last seen at; each turn the surfacer folds the current count, compares to the snapshot, and for **each changed signal** emits a delta carrying the change AND its configured action — e.g. `[delta] inbound gestures 2->3 — one new · run /arc-intake to read bdo's intent`. Unchanged signals stay silent; SessionStart still shows the full level; per-turn is delta-only. The snapshot is a pure cache: deleting it re-derives, the first post-rebuild read baselines to current and surfaces nothing as spuriously new, and a stale snapshot disagreeing with a fresh fold reconciles to the fold, never the snapshot. Fail-open and bounded like every hook (no gh/auth/slow call -> silent, exit 0, never gates a turn).
