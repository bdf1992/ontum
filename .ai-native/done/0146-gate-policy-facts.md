# Done-line 0146 — the value gate judges deterministically-composed policy facts

Written before code, per §9.4. When this line is met, stop. bdo's principle this session: "Judgement should be non-deterministic but the data being judged should be compiled and composed from configuration and policy." The value gate already honors most of this — `compose()` is a deterministic fold that builds the judge's input from the versioned §7 node prompt (policy), the arc membership computed FROM the epic records (not the atom's self-claim), and the prior receipts (the log); the only non-determinism is the model's judgment. The gap: the atom's SELF-CLAIMS (its `incidence.serves`) are still handed to the judge as raw JSON beside the composed truth, so reconciling "the atom asserts X / policy composes Y" is left to the model's reading rather than being a composed, deterministic policy fact — exactly the seam where the recent panel fault lived (an atom claimed an arc no epic confirms).

This amends the gate so EVERY fact the judge weighs is deterministically composed from config + policy, isolating non-determinism to the value call. It does not touch what verdicts a gate may return, the prompt lineage, the trust-rail, or the one-pen write (D-4). Scope guard (§10): the policy-facts block reports composed truth; it never renders a verdict — that stays the model's non-deterministic call.
