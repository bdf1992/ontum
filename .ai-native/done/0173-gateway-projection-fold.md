## Why

epic.diagram's wave-3 cell (`atom.diagram-projection.v0`): point the diagram
organ at the repo so an architecture diagram is a FOLD over truth that cannot
drift. The hard rule (diagrams/CLAUDE.md, the epic context): a from-truth
projection folds INTO `causality/term_economy.py`, never a parallel
`diagram_economy.py` (§10, no double-build). The demonstrating subject is the
gateway topology — the per-directory gateway stack the repo keeps citing — turned
into a deterministic diagram spec whose realness is DERIVED from records (each
layer classified by which evidence resolves), never hand-asserted. The teeth are
term_economy's own: a citation that resolves nowhere is a gap, and the diagram
drops it rather than drawing a lie.

  `classify` and the projection's `gaps` — no second realness derivation.

- The §10 test in `tests/test_diagram.py`: byte-determinism on a fixed fixture,
  and a non-vacuous ghost tooth (a fixture node whose citation resolves nowhere
  is dropped and reported as a gap).

## Not in scope (named, not invented away)

- **A parallel module** — explicitly forbidden; the fold lives in
  `term_economy.py` (§10).
- **A new graph engine or renderer** — `diagrams/compose.py` renders; this fold
  only emits the spec it eats.
- **Auto-layout / a constraint solver** — positions are explicit, computed in
  code from a declared region + tier (the tier-declaration grid-snap aid), the
  byte-deterministic grain; no Sugiyama/router.
- **Promotion of any term past its evidence-derived class** — the cut stays
  bdo's (D-4); the fold classifies and refuses, it never mints.
- **The other epic.diagram cells** (composition/nesting, attribute model,
  generative pole, served experience) — later waves.
