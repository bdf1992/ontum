>
> And the gate has teeth (§10): a passing `tests/test_term_economy.py` proves
> the audit can *refuse*, failing on a fabricated/constant classifier — (a) a
> seeded term with no resolvable evidence classifies as `proposed` /
> `poetic` / `orphaned` / `ghost`, never `minted`; (b) a term whose claimed
> code-evidence `file:line` does not exist on disk is flagged `ghost`; (c) a
> term carrying two incompatible meanings surfaces as `overloaded` (proven on
> the real `seam` and `arc` overloads already on the record); (d) the example
> projection is reproducible from committed input — re-running the tool over
> the seed yields the committed projection byte-for-byte.

## Out of scope, named

- **Building the Causality app / canvas.** No live HTTP API, no browser
  canvas, no persistence DB, no virtual request nodes — those are
  `epic.causality-surface`'s later waves. This increment is the
  terminology-economy witness slice: audits, contracts, one skill, one
  read-only tool, one reproducible projection. The Mermaid render is the only
  visual, and it is text, committed, and deterministic.
- **A master ontology / `terms.json` as truth.** The seed is declared INPUT a
  human or agent authors; the repo (doctrine, code, log, glyph + `language/`
  surfaces) stays truth; the tool only resolves and classifies. No global
  glossary every agent must load.
- **Auto-minting.** The integrator PROPOSES candidate terms; a term is never
  minted without resolved evidence, and promotion past `proposed` stays a
  human/owner judgment (D-4) — the skill refuses to stamp.
- **Folding to main.** Lands as a PR for bdo's validation → his arc-confirm
  gesture → the merge-node lands it; this session does not push to main.
