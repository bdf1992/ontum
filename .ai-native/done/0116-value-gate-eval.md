# Done-line 0116 — The value-gate eval — charades: matched-variant atoms judged by the room

Written before code, per §9.4. When this line is met, stop.

The diagnosis (this session, 2026-06-18, with bdo): an /insights crawl showed the gates refuse only ~3.4% of the time — by §10, a check that never refuses is not checking — and the value-gate prompt names the debt itself ("Semantic evals... are owed"). We have no evidence the L0 gate CAN say no, only that it rarely does. bdo's design (the charades frame): don't ask one judge one question. Give each adversarial case three turns — variants along one axis — and judge each by the reaction of the WHOLE ROOM (a panel of independent judges, not a solo verdict). The reject bar should sit where a fine atom would sometimes be refused; the healing fold (done-line 0114) now catches the over-bites.

The frame (the story-benchmark shape bdo blessed — matched pairs = cheap differential geometry; read-only on truth, D-4): a committed adversarial corpus where each case is a CHEF with three TURNS — a matched-variant axis (plain bad atom -> dressed-up -> the fluent-but-wrong edge, the surface_trap) — plus a genuine accept control so the eval fails a paranoid gate (rejects the control) as well as a soft one (waves a should-reject through). The corpus is DECLARED eval input, never minted onto the real pipeline (like causality's seed.json). Each variant is performed to THE ROOM: a panel of N independent dry judges (no loop.node receipt, no real atom), the verdict being the room's reception — majority, with the spread recorded. Panel size is a parameter, not a constant (the room is bdo's to resize). The scorer is deterministic and reusable for any gate's corpus.

> **Done when:** `loop/gate_eval.py` is a stdlib harness with (a) a committed adversarial corpus of >=4 cases spanning `reject_no_value`, `reject_wrong_value`, `amend`, and >=1 genuine `accept` control, each reject-case carrying a matched-variant axis of 3 turns (escalating toward the fluent-but-wrong edge), each case/variant an atom-shaped stub (story + briefing + incidence + desired_state) with its expected verdict and the one failure mode it tests; (b) a `score(cases, transcript)` that reduces a panel's per-variant verdicts to the room's reception (majority + recorded spread) and computes per-case a robustness curve across the axis plus a pass/fail where the room waving through any should-reject variant FAILS, rejecting the control FAILS, and a gate that catches turn 1 but flips on a later dressed-up turn is flagged BRITTLE; (c) a CLI that renders the corpus read-only and scores a provided panel transcript, ending `done | report`. `tests/test_gate_eval.py` joins the suite with §10 teeth: an all-accept room transcript FAILS the eval (kills a rubber-stamp), an all-reject transcript FAILS the control (kills a paranoid gate), a brittle transcript (catches turn 1, misses turn 3) is flagged brittle, a discriminating+robust transcript passes, and the corpus is well-formed (every case a labeled axis with expected verdicts in the seam's terminal set). The live gate (value-gate.claude.v1) is run by an independent panel over at least one case's full 3-turn axis and its real reception/robustness recorded in the PR. The full suite is green and the work is a PR the merge-node can land.

> **Non-example:** minting the adversarial atoms onto the real pipeline or writing their verdicts through `loop.node judge` (they are declared eval input — and a dry-judge spawn must NOT carry the node-act signature, so the spawn rail reads it as a plain helper, not a branded node-fill); a panel of one (a solo verdict is not the room's reception — the whole point is the room); a scorer with no teeth — one that passes a rubber-stamp gate, a paranoid gate, or a brittle gate (the §10 test exists to kill all three); judging a case with the same actor that authored it (D-2); claiming a catch-rate without running the real gate prompt (a harness that asserts nothing about the live prompt is theater); or re-tuning the gate prompt as part of this increment — adding the owed evals WITHOUT a prompt change is the debt being paid; any later prompt change pairs with an eval change (§7).
