The diagnosis (this session, 2026-06-16, bdo's direction): the loop grew an instrument for every *present* state — `gaps` (open now), `census` (organ aliveness now), `digest` (one span) — but self-critique across the *whole* history stayed a hand-run chat exercise. Patterns that no single span or session reveals (we keep re-deriving the same atom; the cool valve advertised but never opened; a divergence that recurs un-reconciled) slip through, because nothing folds the accumulated record looking for them. bdo: author a refinement-and-retrospective node that runs like the digest, reads our own history, and surfaces the recurring patterns to improve on. The memory note already named the hole: shame senses *stalling*; nothing senses *churn*.

The frame (no double-build, §10): retro reads truth, it is not a second one. It folds the same append-only log every other organ folds (`loop.reconcile.Fold`) and *reuses* the digest's divergence detection (`loop.digest`) rather than re-deriving it — its new axis is **across the whole history**, where the digest is one span. Evidence is cited to log records (the grip rule: a finding points at records, never at prose), it is read-only (it names; the cut and the fix stay a session's or bdo's), and it ends with a clear result like its siblings.

> **Done when:** `loop/retro.py` is a pure, stdlib, read-only fold over the entire log that surfaces recurring patterns as findings, each carrying `kind · subject · cited log evidence · trend/frequency · the one refinement move` (the gaps/digest grain). It implements at least three detectors — **churn** (an atom re-derived ≥2× via version restarts and/or repeated refusals before it landed), **dead-valve** (a control path advertised but never taken — e.g. cool-ticks/total across all ticks at/near zero), and **standing-divergence** (a `digest` divergence that has persisted un-reconciled across the history, carrying its age) — the third reusing `loop.digest`'s divergence detection, not rebuilding it. `python -m loop.retro` renders a scannable retrospective and ends `done | report`; `--json` emits the dataset. A §10 test (`tests/test_retro.py`) joins the suite with teeth: it plants a churn pattern (an atom carrying two refusals before a landing) into a fixture log and asserts the fold catches it, asserts a clean fixture surfaces none, and fails a fabricated/constant classifier (a detector that returns findings without resolving real records). The full suite is green and the work is a PR the merge-node can land.

> **Non-example:** folding report *prose* with regex to mine "insights" (fragile, and against the grip rule — evidence is records, not sentences); re-implementing the digest's span fold or its divergence math here (reuse it — this is the cross-history axis, not a second copy); any write path (retro names patterns; it never judges, admits, dials, or edits the log — D-4); a detector whose finding is a constant or a vibe with no resolvable record behind it (the §10 test exists to kill exactly that); or claiming this single increment *is* the whole refinement system (it is the first node — the fold and its first three detectors; more detectors and any owner surface ride later increments).
