# Done-line 0127 — The continue-beat: ambient auto-continuation that stays silent on escalation

**Goal:** bdo, 2026-06-18 — "can we add an ambient trigger which reminds agents to continue working… sometimes I'm here but I got AFK and I want something to encourage agents to just continue working and stopping if and when they need me, and if the request is escalate then they stop the reminders? Otherwise I don't need to prompt and prompt." Confirmed via the `/ask` panel: build the **in-session** beat now (a `Stop` hook — it continues a live session, it cannot wake a dead one; the timed cloud-wake is a separate, deferred layer); silence the patrol on **needs-you / parked-at-your-stamp / explicit-escalate**, plus a **ceiling that then surfaces**; an empty backlog does **not** silence it (his deliberate un-check — the idle default is to find the next work, never wait for direction; the ceiling is the backstop). **Arc:** serves `epic.landing-throughput-response` — the session-level instance of that epic's "silent patrol, no bureaucracy" principle (`atom.…patrol-stays-silent-no-bureaucracy`); it composes `loop.gaps`/`loop.summon`, it does not double-build them (§10).

Written before code, per §9.4. When this line is met, stop.

> **Done when:** A continue-beat exists and is wired such that: **(1)** `loop/patrol.py` holds a pure read-only `decide(root, session_id, payload)` fold + a write-once `escalate` verb. `decide` returns `("continue", instruction)` when the field has self-serviceable work **and** this session has not escalated **and** the beat count is under the ceiling; it returns `("silent", reason)` when — this session raised the escalate marker, **or** the beat ceiling is reached, **or** the only work left is bdo's stamp (parked-at-stamp: `owner_backlog` non-empty **and** `top_gap` is `None`). It **imports** `loop.gaps.top_gap` and `loop.summon.owner_backlog` (I-4, one definition — never re-derived). The `escalate` verb writes a **session-scoped** marker to the **gitignored** nag-state `.ai-native/continue-beat.json` (the `mock-shame.json` precedent), never the truth log. **(2)** `.claude/hooks/continue_beat.py` is a `Stop` hook that reads the hook JSON from stdin (`session_id`, `stop_hook_active`), prints `{"decision":"block","reason":<instruction>}` to continue (the instruction carries the top gap, the beat N/ceiling, and the one-line `escalate` command), or stays silent (exit 0, no block) to let the agent stop. It is **fail-SAFE to STOP**: any internal error or malformed payload lets the agent stop — the *opposite* direction from the other beats' fail-open-and-continue, named as such, because a bug that always-continued would trap an AFK session burning tokens. **(3)** `settings.json` wires `continue_beat.py` as a `Stop` hook beside `reflect_auto.py`. **(4)** `tests/test_patrol.py` proves the §10 teeth: a *constant* "continue" decider FAILS the escalate, ceiling, and parked-at-stamp cases, and a *constant* "silent" decider FAILS the has-work case — the decision must be derived, not asserted.

> **Non-example:** a hook that continues on its own internal error (fail-open the wrong way — it traps an AFK session burning tokens); silencing globally whenever `owner_backlog` is non-empty (a session can still work non-stamp gaps — silence only when the *only* remaining work is bdo's); writing the escalate marker or the beat count to the truth log (it is ephemeral nag-state, gitignored like `mock-shame.json`); a ceiling that can never be reached (unbounded burn); re-implementing `top_gap`/`owner_backlog` instead of importing them (a twin that drifts, violates I-4).

> **Out of scope — named (next increment, its own done-line):** the **timed cloud-wake layer** (a scheduled routine that wakes a *fresh* session — a `Stop` hook cannot resurrect a process that exited); graduating the beat **ceiling** from a code constant to an **admitted dial** (`--by bdo`, the setpoint shape); **automatic** needs-you detection from mid-session signals (today realized by the agent raising the escalate marker — the honest limit of a `Stop` hook).

> **Evidence expected:** the three files above + the `settings.json` wiring; `tests/test_patrol.py` green inside the full suite; a backed atom with an independent value-gate receipt (D-2); a numbered session report; a merge receipt.
