# Done-line 0078 — loop.field folds the complete environment into one routed, re-derivable typed graph

> **Goal:** goal.holonic-surface

The replication organ of the distributed-holonic-system direction: today
`loop.field` folds **one** arc at a time (`--arc <epic>`), so the loop can
see a single ladder but never the whole. bdo's goal-line is to load the
**complete** ontum environment as one navigable, holographic, jective node
graph — new agenda/arc/epic initiatives appearing as jective casts traveling
through jective nouns, nouns composing into nodes. This done-line builds the
fold that loads the whole; the Causality app that *projects* it is later
work under the same goal (epic.causality-surface), not this line.

It extends the already-built, tested `loop/field.py` (done-line 0050,
`atom.field-topology.v0`) under the confirmed `epic.the-field`. Read-only,
stdlib, no browser — a pure fold over records already on disk. It does NOT
double-build anything: the fold lives here in `loop/`; Causality only ever
reads it (causality/CLAUDE.md). The agenda 7-type schema
(Agenda·Arc·Epic·Story·Task·Goal·Done) stays deferred to bdo's structure
confirm — tonight nodes are typed by what is already first-class on disk
(arc, epic, story, node) and absent rungs are surfaced as gaps, never
invented (the field's existing discipline).

> **Done when:** `loop.field` can fold every confirmed arc into ONE typed
> graph (a `--all` mode, or equivalent, alongside the existing `--arc`),
> the holonic whole — and (a) it composes across at least two epics into a
> single graph that PRESERVES the arc -> epic -> story -> ... -> occupant
> altitude ladder rather than flattening to a bag of nodes; (b) every
> emitted node carries a resolvable route (a record id on the log or a
> file:line on disk, never prose) — a node with no route is surfaced as a
> gap/ghost, never emitted as fact; (c) the whole projection is
> re-derivable byte-for-byte from the same input; and a test in `tests/`
> asserts all three.

> **Non-example:** a flattened list of all nodes with the altitude ladder
> collapsed (the unreadable hairball the prototype named as make-or-break);
> OR any node emitted without a resolvable route (a ghost the holographic
> rule forbids); OR a fold that is not byte-reproducible (a reflection that
> drifts, not a projection).

> **Pressure reduced:** replication — the one missing organ of the
> four-fold self (recursion / improvement / communication already have
> organs). Closes the literal "load the complete ontum environment" gap and
> lays the foundation epic.the-field's portal and epic.causality-surface's
> projection both render.

> **Does not complete:** Causality the app (persist / config / ingest /
> project / API — epic.causality-surface); the signal/occupancy layer
> (atom.signal-layer.v0); any visual portal; OP3 (pressure composes); and
> the agenda 7-type schema (deferred to bdo's structure confirm). Those are
> continuing pressure under goal.holonic-surface, not this line.

> **Evidence expected:** the extended `loop/field.py`; a passing test in
> `tests/` proving whole-environment composition across >=2 epics, a route
> on every emitted node, and byte-identical re-fold; CLI output showing the
> composed whole-field graph.
