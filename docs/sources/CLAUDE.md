# docs/sources/ — inspiration and direction, read-only

External sources aligned with the work: things to return to **when we hit
walls or question our sanity**. Like `docs/phase-2/`, these are **read-only
context, not material** — the loop must not import, vendor, or build from
anything here. A source justifies a *direction*; it never specifies a build.
If a build seems to need one of these, that is a scope error — report it,
don't code around it (root hard rule).

This file is governance, not a source — it lets a session *file* a new
source entry (write_guard requires a governing `CLAUDE.md`). It is not
`@`-imported by the root surface: its knowledge binds only while working
here, so it loads by nesting (root CLAUDE.md's composition rule).

## Form of an entry

One markdown file per source, plus a one-line pointer in `README.md`. Each
entry separates the inspiration from the claim, in these sections:

- **0. The source** — what it is, where it came from, who surveyed it and
  when, with the read-only reminder.
- **1. What the source says** — the claim stated plainly, in its own terms.
- **2. Why it is aligned** — the rhyme with ontum's real surfaces, and the
  divergence worth keeping in view (the §10 instinct: name where it differs,
  not only where it agrees).
- **3. The direction it points** — what wall this is filed against, and what
  it justifies. Never a build spec.

A source is never minted into doctrine by appearing here; promotion of any
direction past "inspiration" stays the loop's work and bdo's stamp (D-4).
