# Cube-Coil Reader Key

Teaches one move: **how the system computes a term**, and **how a human reads one
once computed.** Don't visualize the whole lexicon — visualize the *act of reading* it.

## The read (the surface — what a human asks of a *computed* term)

> **PLACE · FIELD · PATH · STATUS**

| reader question | answer | note |
|---|---|---|
| **Where is it?** (PLACE) | center / face / vertex / edge | 1 / 6 / 8 / 12 |
| **What does it carry?** (FIELD) | charge · color · flavor · spin · mass · parity | |
| **How does it move?** (PATH) | helical coil · winding · neck-crossing · inversion | PATH = the *trace*, computed — never primitive |
| **How trusted is it?** (STATUS) | PINNED · OPEN · MYTHOS | only PINNED may score |

## The point

A reader doesn't need every physics/math analogy first. They need: **the path is
computed (a trace), coupling is earned (emergent), and only grounded (PINNED) terms
may score.** Mythos orients but never computes. That's the whole reading discipline
on one card.
